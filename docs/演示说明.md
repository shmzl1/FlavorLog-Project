本文档用于课堂演示前准备和现场演示流程说明。

---

## 1. 演示环境

| 项目     | 说明                        |
| -------- | --------------------------- |
| 前端     | Flutter App                 |
| 后端     | FastAPI                     |
| 数据库   | PostgreSQL 16               |
| 部署方式 | 局域网 C/S                  |
| 演示设备 | 一台电脑 + 一台手机或模拟器 |

## 2. 演示前准备

### 2.1 启动数据库

在项目根目录执行：

```bash
docker compose up -d
```

检查容器：

```bash
docker ps
```

如果数据库启动失败，查看日志：

```bash
docker logs flavorlog-postgres
```

---

### 2.2 启动后端

进入后端目录：

```bash
cd backend
```

如果还没有创建虚拟环境，先创建：

```bash
python -m venv .venv
```

Windows PowerShell 激活虚拟环境：

```powershell
.venv\Scripts\activate
```

macOS / Linux 激活虚拟环境：

```bash
source .venv/bin/activate
```

安装依赖：

```bash
pip install -r requirements.txt
```

启动后端服务：

```bash
uvicorn main:app --reload --host 0.0.0.0 --port 8000
```

浏览器访问 Swagger：

```text
http://127.0.0.1:8000/docs
```

健康检查接口：

```text
http://127.0.0.1:8000/health
```

局域网健康检查接口：

```text
http://电脑局域网IP:8000/health
```

---

### 2.3 启动前端

进入前端目录：

```bash
cd frontend
```

安装依赖：

```bash
flutter pub get
```

检查 Flutter 环境：

```bash
flutter doctor
```

运行前端：

```bash
flutter run
```

## 3. 真机联调注意事项

1. 手机和电脑必须连接同一个 Wi-Fi。
2. 后端必须使用 `--host 0.0.0.0` 启动。
3. 前端接口地址不能写 `127.0.0.1`，要写电脑局域网 IP。
4. Windows 防火墙可能拦截 8000 端口，需要允许访问。
5. 真机调试前，先在手机浏览器中访问后端健康检查接口，确认手机能访问电脑后端。

示例：

```text
http://192.168.1.100:8000/health
```

其中 `192.168.1.100` 需要替换为后端电脑的真实局域网 IP。

## 8. 常见问题处理

### 8.1 手机访问不到后端

检查：

1. 手机和电脑是否在同一个 Wi-Fi。
2. 后端是否用 `0.0.0.0` 启动。
3. 前端是否使用电脑局域网 IP。
4. Windows 防火墙是否拦截 8000 端口。
5. 手机浏览器是否能打开 `http://电脑局域网IP:8000/health`。

---

### 8.2 Swagger 打不开

检查后端是否启动成功：

```bash
uvicorn main:app --reload --host 0.0.0.0 --port 8000
```

然后访问：

```text
http://127.0.0.1:8000/docs
```

---

### 8.3 数据库连接失败

检查数据库容器：

```bash
docker ps
```

检查数据库日志：

```bash
docker logs flavorlog-postgres
```

必要时重建数据库：

```bash
docker compose down -v
docker compose up -d
```

---

### 8.4 前端依赖安装失败

进入前端目录后执行：

```bash
flutter clean
flutter pub get
```

然后重新运行：

```bash
flutter run
```

## 11. 最终演示目标

阶段一演示的核心不是展示完整高并发架构，而是展示“知味志”的核心业务闭环：

```text
多模态饮食数据采集
    ↓
结构化饮食记录
    ↓
赛博冰箱食材管理
    ↓
AI / Mock 食谱推荐
    ↓
餐后健康反馈
    ↓
红黑榜 / 健康报告
    ↓
社区分享与互动
```

只要该闭环能够在局域网环境中跑通，就达到阶段一 MVP 演示目标。
