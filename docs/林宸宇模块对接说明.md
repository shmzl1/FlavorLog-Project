## 已实现接口

### 健康反馈

- `POST /health/`
  - 新增餐后健康反馈。
  - 请求体沿用 `HealthFeedbackCreate`。

- `GET /health/`
  - 获取当前用户全部健康反馈，兼容原有接口。

- `GET /health/feedbacks`
  - 支持分页和日期筛选。
  - 参数：`page`、`page_size`、`start_date`、`end_date`。

### 食物红黑榜

- `GET /health/blacklist`
  - 从当前用户饮食记录和餐后反馈中分析黑榜、红榜。
  - 如果课堂演示时没有真实数据，会返回结构一致的 Mock 数据。

### 健康周报

- `GET /health/weekly-report`
  - 参数：`week_start`，格式 `YYYY-MM-DD`，可不传。
  - 返回本周平均热量、平均蛋白质、每日热量趋势、提醒和建议。

### 口味搭子

- `POST /community/taste-buddies/match`
  - 请求体：

```json
{
  "top_k": 5,
  "prefer_same_goal": true
}
```

  - 会根据饮食记录生成当前用户口味向量，并用余弦相似度匹配其他用户。
  - 如果没有候选用户，会返回结构一致的 Mock 搭子数据。

## 课堂演示流程

### 1. 启动后端

如果 8000 端口已经被旧服务占用，建议用 8001 启动当前代码：

```powershell
cd C:\Users\HP\Desktop\软工\FlavorLog-Project-main\backend
conda activate flavorlog
uvicorn main:app --reload --host 127.0.0.1 --port 8001
```

浏览器打开：

```text
http://127.0.0.1:8001/docs
```

### 2. 登录授权

在 Swagger 右上角点 `Authorize`，填写：

```text
username: lin_demo@flavorlog.local
password: 123456
```

### 3. 演示健康反馈

打开 `Health` 分组：

- `GET /api/v1/health/feedbacks`
- 参数可填：`page=1`、`page_size=10`
- 展示当前用户餐后健康反馈列表

也可以演示新增反馈：

```json
{
  "bloating_level": 7,
  "fatigue_level": 6,
  "mood": "low",
  "digestive_note": "餐后有明显腹胀",
  "extra_symptoms": ["bloating"],
  "food_record_id": 1
}
```

### 4. 演示食物红黑榜

调用：

```text
GET /api/v1/health/blacklist
```

重点讲：

- 黑榜根据饮食记录和餐后不适反馈生成
- 红榜根据负反馈较少、营养结构较友好生成
- 没有真实数据时也会返回 Mock 结构，保证课堂演示不断档

### 5. 演示健康周报

调用：

```text
GET /api/v1/health/weekly-report
```

可以不填 `week_start`，也可以填类似：

```text
2026-05-11
```

展示内容：

- 本周平均热量
- 本周平均蛋白质
- 每日热量趋势
- warnings 和 suggestions

### 6. 演示口味搭子

打开 `Community` 分组，调用：

```text
POST /api/v1/community/taste-buddies/match
```

请求体：

```json
{
  "top_k": 5,
  "prefer_same_goal": true
}
```

重点讲：

- 系统从饮食记录里生成口味标签和向量
- 用余弦相似度匹配相似用户
- `prefer_same_goal=true` 时，同健康目标用户会有轻微加权

## 和其他成员对接

### 和刘子恒前端对接

健康报告页面可以直接调用：

- `GET /health/feedbacks`
- `GET /health/blacklist`
- `GET /health/weekly-report`

社区页面或搭子入口调用：

- `POST /community/taste-buddies/match`

前端字段按接口文档返回结构展示即可，红黑榜和搭子接口都已经保证没有真实数据时也能返回演示数据。

### 和吴越饮食记录模块对接

红黑榜和周报依赖：

- `food_records.total_calories`
- `food_records.total_protein_g`
- `food_record_items.food_name`
- `health_feedbacks.food_record_id`

新增健康反馈时，前端最好把对应的 `food_record_id` 一起传过来，这样红黑榜分析更准确。

### 和焦思源推荐模块对接

推荐模块可读取或复用：

- `taste_vectors.vector_json`
- `taste_vectors.tags`
- `GET /health/blacklist` 的 `black_items` 和 `red_items`

建议推荐时把黑榜食材加入 `avoid_ingredients`，把红榜食材作为 `preferred_ingredients` 的参考。

### 和唐丞杰社区模块对接

口味搭子接口挂在社区模块：

- `POST /community/taste-buddies/match`

社区动态 Mock 数据会生成带标签的帖子，可用于课堂演示点赞、评论、搭子匹配入口。
