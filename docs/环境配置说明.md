```text
数据库：Docker Desktop + Docker Compose 启动 PostgreSQL
后端 Python 环境：Miniconda / conda
前端环境：Flutter stable
后端框架：FastAPI
数据库：PostgreSQL 16
```

## 4. 根目录环境变量配置

根目录 `.env.example` 是项目级环境变量模板。

Windows PowerShell：

```powershell
copy .env.example .env
```

macOS / Linux：

```bash
cp .env.example .env
```

注意：

- `.env` 是本地私有配置文件。
- `.env` 不能提交到 GitHub。
- 如果 `.env.example` 改了，组长需要通知所有组员同步更新自己的 `.env`。
- 复制文件时必须保留换行和缩进，不能把文件内容压成一行。

## 5. 使用 Docker Desktop 启动数据库

本项目使用 Docker Desktop + Docker Compose 启动 PostgreSQL。

启动前必须确认：

1. Docker Desktop 已经打开。
2. Docker Desktop 左下角显示 Engine 正在运行。
3. 当前命令行位于项目根目录。
4. 根目录已经存在 `.env` 文件。

在项目根目录执行：

```bash
docker compose up -d
```

查看容器状态：

```bash
docker ps
```

如果看到类似下面的容器，说明数据库启动成功：

```text
flavorlog-postgres
```

查看数据库日志：

```bash
docker logs flavorlog-postgres
```

停止数据库：

```bash
docker compose down
```

删除数据库数据并重新初始化：

```bash
docker compose down -v
docker compose up -d
```

## 6. 数据库连接信息

当前项目统一使用以下数据库配置：

| 项目           | 值                 |
| -------------- | ------------------ |
| 数据库名       | flavorlog          |
| 用户名         | flavorlog_user     |
| 密码           | flavorlog_password |
| 容器内端口     | 5432               |
| 宿主机访问端口 | 5433               |

后端从本机连接 Docker 中的 PostgreSQL 时，使用：

```text
postgresql+psycopg2://flavorlog_user:flavorlog_password@localhost:5433/flavorlog
```

注意：

- `5432` 是 PostgreSQL 容器内部端口。
- `5433` 是电脑宿主机访问 Docker 数据库时使用的端口。
- 如果你电脑本机没有安装 PostgreSQL，也可以把 `.env` 和 `docker-compose.yml` 中的宿主机端口改成 `5432`。
- 端口修改必须同时修改 `.env`、`backend/.env` 和 `docker-compose.yml`。

## 7. 后端环境配置：Miniconda

本项目后端使用 Miniconda 创建 Python 虚拟环境。

### 7.1 创建 conda 环境

进入后端目录：

```bash
cd backend
```

创建环境：

```bash
conda create -n flavorlog python=3.11 -y
```

激活环境：

```bash
conda init powershell
conda activate flavorlog
```

如果激活成功，命令行前面会出现：

```text
(flavorlog)
```

### 7.2 安装后端依赖

确保当前目录是 `backend/`：

```bash
pip install -r requirements.txt
```

### 7.3 创建后端环境变量文件

Windows PowerShell：

```powershell
copy .env.example .env
```

macOS / Linux：

```bash
cp .env.example .env
```

后端 `.env` 中的数据库连接应为：

```text
DATABASE_URL=postgresql+psycopg2://flavorlog_user:flavorlog_password@localhost:5433/flavorlog
```

### 7.4 启动后端

确保当前目录是 `backend/`，并且已经激活 conda 环境：

```bash
conda activate flavorlog
uvicorn main:app --reload --host 0.0.0.0 --port 8000
```

启动成功后访问：

```text
http://127.0.0.1:8000
```

Swagger 接口文档：

```text
http://127.0.0.1:8000/docs
```

ReDoc 接口文档：

```text
http://127.0.0.1:8000/redoc
```

健康检查接口：

```text
http://127.0.0.1:8000/health
```

## 9. 局域网真机联调配置

阶段一需要手机和电脑处于同一个局域网。

### 9.1 查看电脑局域网 IP

Windows PowerShell：

```powershell
ipconfig
```

找到 WLAN 或以太网中的 IPv4 地址，例如：

```text
192.168.1.100
```

### 9.2 后端启动方式

后端必须使用：

```bash
uvicorn main:app --reload --host 0.0.0.0 --port 8000
```

不能只绑定 `127.0.0.1`，否则手机访问不到。

### 9.3 手机端后端地址

前端中后端基础地址必须配置为：

```text
http://电脑局域网IP:8000/api/v1
```

例如：

```text
http://192.168.1.100:8000/api/v1
```

不要在手机真机联调时写：

```text
http://127.0.0.1:8000/api/v1
```

因为手机上的 `127.0.0.1` 指的是手机自己，不是后端电脑。

## 10. 前后端联调约定

### 10.1 接口基础地址

本地模拟器：

```text
http://127.0.0.1:8000/api/v1
```

手机真机：

```text
http://电脑局域网IP:8000/api/v1
```

### 10.2 所有接口统一返回格式

成功：

```json
{
  "code": 0,
  "message": "success",
  "data": {},
  "request_id": "uuid-string",
  "timestamp": "2026-05-06T12:00:00+08:00"
}
```

失败：

```json
{
  "code": 40001,
  "message": "参数错误",
  "data": null,
  "errors": [
    {
      "field": "email",
      "reason": "邮箱格式不正确"
    }
  ],
  "request_id": "uuid-string",
  "timestamp": "2026-05-06T12:00:00+08:00"
}
```

### 10.3 鉴权方式

登录成功后，后端返回 `access_token`。

前端后续请求必须在 Header 中携带：

```text
Authorization: Bearer <access_token>
```

### 10.4 时间格式

所有时间统一使用 ISO 8601 字符串：

```text
2026-05-06T12:00:00+08:00
```

### 10.5 ID 格式

阶段一所有主键 ID 统一使用整数类型。

后续如果需要 UUID，必须先修改接口文档和数据库设计文档，再改代码。

## 11. 数据库初始化

数据库初始化 SQL 文件放在：

```text
backend/database/init/init.sql
```

首次执行：

```bash
docker compose up -d
```

PostgreSQL 容器会自动执行该目录下的 SQL 文件。

如果已经启动过数据库，修改了 `init.sql` 后不会自动重新执行。需要执行：

```bash
docker compose down -v
docker compose up -d
```


## 13. 常见问题

### 13.1 后端启动失败：找不到模块

确认当前目录是：

```text
FlavorLog-Project/backend
```

并确认 conda 环境已经激活：

```bash
conda activate flavorlog
```

然后执行：

```bash
uvicorn main:app --reload --host 0.0.0.0 --port 8000
```

### 13.2 conda activate 失败

先确认已经安装 Miniconda，并且命令行能识别：

```bash
conda --version
```

如果 PowerShell 无法激活 conda，执行：

```powershell
conda init powershell
```

然后关闭当前 PowerShell，重新打开。

### 13.3 前端手机访问不到后端

检查：

1. 手机和电脑是否在同一 Wi-Fi。
2. 后端是否使用 `--host 0.0.0.0` 启动。
3. 前端接口地址是否使用电脑局域网 IP。
4. Windows 防火墙是否拦截 8000 端口。
5. 手机浏览器是否能打开 `http://电脑局域网IP:8000/health`。

### 13.4 数据库连接失败

检查：

```bash
docker ps
```

确认 `flavorlog-postgres` 容器正在运行。

检查后端 `.env` 中的数据库连接串：

```text
DATABASE_URL=postgresql+psycopg2://flavorlog_user:flavorlog_password@localhost:5433/flavorlog
```

注意这里是 `5433`，不是 `5432`。

### 13.5 修改了数据库初始化 SQL 但没生效

执行：

```bash
docker compose down -v
docker compose up -d
```

### 13.6 YAML / Python / Markdown 文件运行异常

检查文件是否被压成一整行。

以下文件必须保留正常换行和缩进：

```text
docker-compose.yml
frontend/pubspec.yaml
frontend/analysis_options.yaml
backend/main.py
backend/database/init/init.sql
docs/环境配置说明.md
README.md
backend/README.md
```
