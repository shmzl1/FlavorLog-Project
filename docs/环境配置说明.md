# 知味志 FlavorLog 环境配置说明

本文档用于统一所有组员的开发环境。组员必须先按照本文档完成环境配置，再开始写代码。

---

## 1. 项目技术栈

### 1.1 前端

- Flutter
- Dart
- GetX：状态管理与路由管理
- Dio：HTTP 请求封装
- SQLite / sqflite：本地离线缓存
- Camera / Image Picker / Permission Handler：拍照、视频、权限申请

### 1.2 后端

- Python 3.11
- FastAPI
- Uvicorn
- Pydantic
- SQLAlchemy
- PostgreSQL
- python-dotenv

### 1.3 数据库

- PostgreSQL 16
- Docker Compose 管理数据库容器

### 1.4 阶段二预留

阶段一暂时不强制实现，但目录和接口设计需要预留：

- Redis
- Celery
- RabbitMQ / Redis List
- PgBouncer
- WebSocket

## 5. 启动数据库

本项目使用 Docker Compose 启动 PostgreSQL。

在项目根目录执行：

```bash
docker compose up -d
```

查看容器状态：

```bash
docker ps
```

查看日志：

```bash
docker logs flavorlog-postgres
```

停止数据库：

```bash
docker compose down
```

删除数据库数据并重新初始化：

```bash
docker compose down -v
docker compose up -d
```

数据库默认信息：

| 项目     | 值                 |
| -------- | ------------------ |
| 数据库名 | flavorlog          |
| 用户名   | flavorlog_user     |
| 密码     | flavorlog_password |
| 主机     | localhost          |
| 端口     | 5432               |

本地连接串：

```text
postgresql://flavorlog_user:flavorlog_password@localhost:5432/flavorlog
```

## 8. 局域网真机联调配置

阶段一需要手机和电脑处于同一个局域网。

### 8.1 查看电脑局域网 IP

Windows PowerShell：

```powershell
ipconfig
```

找到 WLAN 或以太网中的 IPv4 地址，例如：

```text
192.168.1.100
```

### 8.2 后端启动方式

后端必须使用：

```bash
uvicorn main:app --reload --host 0.0.0.0 --port 8000
```

不能只绑定 `127.0.0.1`，否则手机访问不到。

### 8.3 手机端后端地址

前端中后端基础地址必须配置为：

```text
http://电脑局域网IP:8000/api/v1
```

例如：

```text
http://192.168.1.100:8000/api/v1
```

不要在手机真机联调时写：

```text
http://127.0.0.1:8000/api/v1
```

因为手机上的 `127.0.0.1` 指的是手机自己，不是后端电脑。

## 9. 前后端联调约定

### 9.1 接口基础地址

本地模拟器：

```text
http://127.0.0.1:8000/api/v1
```

手机真机：

```text
http://电脑局域网IP:8000/api/v1
```

### 9.2 所有接口统一返回格式

成功：

```json
{
  "code": 0,
  "message": "success",
  "data": {},
  "request_id": "uuid-string",
  "timestamp": "2026-05-06T12:00:00+08:00"
}
```

失败：

```json
{
  "code": 40001,
  "message": "参数错误",
  "data": null,
  "errors": [
    {
      "field": "email",
      "reason": "邮箱格式不正确"
    }
  ],
  "request_id": "uuid-string",
  "timestamp": "2026-05-06T12:00:00+08:00"
}
```

### 9.3 鉴权方式

登录成功后，后端返回 `access_token`。

前端后续请求必须在 Header 中携带：

```text
Authorization: Bearer <access_token>
```

### 9.4 时间格式

所有时间统一使用 ISO 8601 字符串：

```text
2026-05-06T12:00:00+08:00
```

### 9.5 ID 格式

阶段一所有主键 ID 统一使用整数类型。

后续如果需要 UUID，必须先修改接口文档和数据库设计文档，再改代码。

## 10. 数据库初始化

数据库初始化 SQL 文件放在：

```text
backend/database/init/init.sql
```

当首次执行：

```bash
docker compose up -d
```

PostgreSQL 容器会自动执行该目录下的 SQL 文件。

如果已经启动过数据库，修改了 `init.sql` 后不会自动重新执行。需要执行：

```bash
docker compose down -v
docker compose up -d
```

## 11. 组员开发流程

每次开始开发前：

```bash
git pull
```

创建自己的功能分支：

```bash
git checkout -b feat/模块名-功能名
```

例如：

```bash
git checkout -b feat/auth-login
git checkout -b feat/food-record-page
git checkout -b feat/fridge-api
```

开发完成后：

```bash
git status
git add .
git commit -m "feat: 完成登录接口"
git push
```

公共文件修改规则：

- `docs/接口文档.md` 只能由组长或经组长确认后修改。
- `docs/数据库设计.md` 只能由组长或后端数据库负责人修改。
- `.env.example`、`docker-compose.yml`、`requirements.txt`、`pubspec.yaml` 修改前必须通知全组。
- 数据库字段、接口字段不能私自改名。

## 12. 常见问题

### 12.1 后端启动失败：找不到模块

确认当前目录是：

```text
FlavorLog-Project/backend
```

然后执行：

```bash
uvicorn main:app --reload --host 0.0.0.0 --port 8000
```

### 12.2 前端手机访问不到后端

检查：

1. 手机和电脑是否在同一 Wi-Fi。
2. 后端是否使用 `--host 0.0.0.0` 启动。
3. 前端接口地址是否使用电脑局域网 IP。
4. Windows 防火墙是否拦截 8000 端口。

### 12.3 数据库连接失败

检查：

```bash
docker ps
```

确认 `flavorlog-postgres` 容器正在运行。

检查后端 `.env` 中的数据库连接串：

```text
DATABASE_URL=postgresql+psycopg2://flavorlog_user:flavorlog_password@localhost:5432/flavorlog
```

### 12.4 修改了数据库初始化 SQL 但没生效

执行：

```bash
docker compose down -v
docker compose up -d
```

## 13. 本阶段必须能跑通的最小链路

阶段一至少要保证以下链路可联调：

1. 前端启动成功。
2. 后端启动成功。
3. PostgreSQL 启动成功。
4. Swagger 能访问。
5. 注册接口能调通。
6. 登录接口能返回 token。
7. 前端能携带 token 调用用户信息接口。
8. 饮食记录能新增、查询。
9. 赛博冰箱食材能新增、查询。
10. 社区动态能发布、查询、点赞。
