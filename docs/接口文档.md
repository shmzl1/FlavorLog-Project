# FlavorLog 接口文档

## 一、接口说明

本文档用于统一前后端接口约定。

阶段一接口以 RESTful API 为主，由 FastAPI 提供服务，Flutter 前端通过 Dio 调用接口。

---

## 二、基础地址

### 本地开发地址

```text
http://127.0.0.1:8000/api/v1
```

### 局域网手机访问地址

手机和电脑需要连接到同一个局域网。

将 `127.0.0.1` 替换为后端电脑的局域网 IP。

示例：

```text
http://192.168.1.100:8000/api/v1
```

---

## 三、接口文档地址

FastAPI 启动后，可访问自动生成的 Swagger 文档：

```text
http://127.0.0.1:8000/docs
```

## 四、统一响应格式

## 1. 成功响应

```json
{
  "code": 200,
  "message": "success",
  "data": {}
}
```

## 2. 失败响应

```json
{
  "code": 400,
  "message": "error message",
  "data": null
}
```

## 3. 分页响应

```json
{
  "code": 200,
  "message": "success",
  "data": {
    "items": [],
    "total": 0,
    "page": 1,
    "page_size": 10
  }
}
```

---

## 五、通用状态码

| 状态码 | 含义                |
| ------ | ------------------- |
| 200    | 请求成功            |
| 201    | 创建成功            |
| 400    | 请求参数错误        |
| 401    | 未登录或 Token 无效 |
| 403    | 无权限              |
| 404    | 资源不存在          |
| 500    | 服务器内部错误      |

---

## 六、鉴权说明

除注册、登录接口外，其他需要用户身份的接口都需要在请求头中携带 Token。

```text
Authorization: Bearer <access_token>
```

---

## 七、用户与鉴权模块

## 1. 用户注册

```text
POST /auth/register
```

### 请求参数

```json
{
  "username": "test_user",
  "password": "123456",
  "nickname": "测试用户"
}
```

### 响应示例

```json
{
  "code": 200,
  "message": "success",
  "data": {
    "user_id": 1,
    "username": "test_user",
    "nickname": "测试用户"
  }
}
```

---

## 2. 用户登录

```text
POST /auth/login
```

### 请求参数

```json
{
  "username": "test_user",
  "password": "123456"
}
```

### 响应示例

```json
{
  "code": 200,
  "message": "success",
  "data": {
    "access_token": "token_string",
    "token_type": "bearer",
    "user": {
      "user_id": 1,
      "username": "test_user",
      "nickname": "测试用户"
    }
  }
}
```

---

## 3. 获取当前用户信息

```text
GET /users/me
```

### 请求头

```text
Authorization: Bearer <access_token>
```

## 八、饮食记录模块

## 1. 新增饮食记录

```text
POST /food/records
```

### 请求参数

```json
{
  "meal_type": "lunch",
  "food_name": "鸡胸肉沙拉",
  "calories": 450,
  "protein": 35.5,
  "fat": 12.0,
  "carbohydrate": 40.0,
  "record_time": "2026-04-28 12:30:00",
  "remark": "午餐"
}
```

---

## 2. 获取饮食记录列表

```text
GET /food/records
```

### 查询参数

| 参数       | 类型   | 是否必填 | 说明     |
| ---------- | ------ | -------- | -------- |
| page       | int    | 否       | 页码     |
| page_size  | int    | 否       | 每页数量 |
| start_date | string | 否       | 开始日期 |
| end_date   | string | 否       | 结束日期 |

---

## 3. 获取单条饮食记录

```text
GET /food/records/{record_id}
```

---

## 4. 删除饮食记录

```text
DELETE /food/records/{record_id}
```

## 十四、接口命名规范

1. 路径使用小写英文。
2. 多单词使用中划线或下划线，优先使用中划线。
3. 接口路径尽量使用名词。
4. 查询类使用 GET。
5. 新增类使用 POST。
6. 修改类使用 PUT。
7. 删除类使用 DELETE。
