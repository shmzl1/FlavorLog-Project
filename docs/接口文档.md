# 知味志 FlavorLog 前后端接口文档

本文档是前端与后端联调的唯一依据。任何接口路径、请求字段、响应字段发生变化，都必须先修改本文档，再修改代码。

---

## 1. 基础约定

### 1.1 接口基础路径

本地开发：

```text
http://127.0.0.1:8000/api/v1
```

局域网真机联调：

```text
http://电脑局域网IP:8000/api/v1
```

示例：

```text
http://192.168.1.100:8000/api/v1
```

### 1.2 Swagger 地址

```text
http://127.0.0.1:8000/docs
```

### 1.3 ReDoc 地址

```text
http://127.0.0.1:8000/redoc
```

---

## 2. 通用请求规范

### 2.1 请求格式

除文件上传接口外，所有请求统一使用 JSON：

```http
Content-Type: application/json
```

文件上传接口使用：

```http
Content-Type: multipart/form-data
```

### 2.2 鉴权方式

需要登录的接口必须携带：

```http
Authorization: Bearer <access_token>
```

### 2.3 时间格式

所有时间字段统一使用 ISO 8601 字符串：

```text
2026-05-06T12:00:00+08:00
```

### 2.4 命名规范

- JSON 字段统一使用小写蛇形命名：`snake_case`
- ID 字段统一命名为：`id`
- 用户 ID 统一命名为：`user_id`
- 创建时间统一命名为：`created_at`
- 更新时间统一命名为：`updated_at`

---

## 4. 通用状态码

|  code | HTTP 状态码 | 含义               |
| ----: | ----------: | ------------------ |
|     0 |         200 | 成功               |
| 40001 |         400 | 参数错误           |
| 40002 |         400 | 业务规则不允许     |
| 40101 |         401 | 未登录             |
| 40102 |         401 | Token 无效或过期   |
| 40301 |         403 | 无权限             |
| 40401 |         404 | 资源不存在         |
| 40901 |         409 | 资源冲突           |
| 50001 |         500 | 服务器内部错误     |
| 50002 |         500 | 第三方服务调用失败 |

# 5. 鉴权模块 Auth

## 5.1 用户注册

```http
POST /auth/register
```

是否需要登录：否

### 请求体

```json
{
  "username": "test_user",
  "email": "test@example.com",
  "password": "123456",
  "nickname": "测试用户"
}
```

### 字段说明

| 字段     | 类型   | 必填 | 说明         |
| -------- | ------ | ---- | ------------ |
| username | string | 是   | 用户名，唯一 |
| email    | string | 是   | 邮箱，唯一   |
| password | string | 是   | 密码         |
| nickname | string | 否   | 昵称         |

### 响应 data

```json
{
  "user": {
    "id": 1,
    "username": "test_user",
    "email": "test@example.com",
    "nickname": "测试用户",
    "avatar_url": null,
    "created_at": "2026-05-06T12:00:00+08:00"
  },
  "access_token": "jwt-token-string",
  "token_type": "bearer"
}
```

# 14. 接口变更规则

任何人需要修改接口时，必须按以下流程：

1. 先在群里说明要改什么。
2. 修改 `docs/接口文档.md`。
3. 前端负责人确认字段够用。
4. 后端负责人确认实现可行。
5. 组长确认后再改代码。
