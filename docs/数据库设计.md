# FlavorLog 数据库设计

## 一、数据库说明

本项目阶段一使用 PostgreSQL 作为主数据库。

数据库主要用于存储：

- 用户信息。
- 饮食记录。
- 冰箱食材。
- 健康反馈。
- 社区动态。
- 评论信息。
- 点赞关系。
- 推荐记录。
- 用户口味向量。
- Mock 演示数据。

---

## 二、数据库基础信息

```text
数据库类型：PostgreSQL
数据库版本：16
数据库名称：flavorlog_db
默认用户名：flavorlog
默认密码：flavorlog123
默认端口：5432
```

本地连接字符串：

```text
postgresql://flavorlog:flavorlog123@localhost:5432/flavorlog_db
```

## 三、命名规范

### 1. 表名规范

表名统一使用小写英文和下划线。

示例：

```text
users
food_records
fridge_items
health_feedbacks
community_posts
post_comments
post_likes
taste_vectors
```

### 2. 字段命名规范

字段名统一使用小写英文和下划线。

示例：

```text
user_id
created_at
updated_at
food_name
record_time
```

### 3. 主键规范

每张表使用自增整数主键：

```text
id
```

### 4. 时间字段规范

建议每张主要业务表都包含：

```text
created_at
updated_at
```

| 字段名         | 类型         | 说明                         |
| -------------- | ------------ | ---------------------------- |

## 五、表关系说明

## 1. 用户与饮食记录

```text
users 1 --- N food_records
```

一个用户可以拥有多条饮食记录。

---

## 2. 用户与冰箱食材

```text
users 1 --- N fridge_items
```

一个用户可以拥有多个冰箱食材。

---

## 3. 用户与健康反馈

```text
users 1 --- N health_feedbacks
```

一个用户可以拥有多条健康反馈。

---

## 4. 饮食记录与健康反馈

```text
food_records 1 --- N health_feedbacks
```

一条饮食记录可以对应多条餐后反馈。

---

## 5. 用户与社区动态

```text
users 1 --- N community_posts
```

一个用户可以发布多条社区动态。

---

## 6. 社区动态与评论

```text
community_posts 1 --- N post_comments
```

一条动态可以有多条评论。

---

## 7. 社区动态与点赞

```text
community_posts 1 --- N post_likes
```

一条动态可以有多个点赞记录。

---

## 六、索引设计建议

阶段一建议添加以下索引：

| 表名             | 字段             | 说明                 |
| ---------------- | ---------------- | -------------------- |
| users            | username         | 加快登录查询         |
| food_records     | user_id          | 加快用户饮食记录查询 |
| food_records     | record_time      | 加快按时间查询       |
| fridge_items     | user_id          | 加快用户冰箱查询     |
| health_feedbacks | user_id          | 加快健康反馈查询     |
| community_posts  | created_at       | 加快社区信息流排序   |
| post_comments    | post_id          | 加快动态评论查询     |
| post_likes       | post_id, user_id | 防止重复点赞         |
| taste_vectors    | user_id          | 加快口味匹配查询     |

PostgreSQL 支持 JSONB 字段，适合存储结构灵活的数据。

本项目中以下字段建议使用 JSONB：

```text
recipe_recommendations.recipe_content
taste_vectors.vector_data
ai_analysis_logs.result_data
```

## 八、初始化 SQL 文件

数据库初始化文件放在：

```text
backend/database/init/init.sql
```

Docker 首次创建 PostgreSQL 容器时，会自动执行该目录下的 SQL 文件。

注意：

```text
只有首次创建数据库数据卷时会自动执行 init.sql。
如果需要重新执行初始化 SQL，需要删除旧数据卷。
```

重置命令：

```bash
docker compose down -v
docker compose up -d
```

## 九、后续维护说明

本数据库设计为阶段一初始版本。

后续如果新增表或字段，需要同步修改：

```text
backend/app/models/
backend/app/schemas/
backend/database/init/init.sql
docs/数据库设计.md
```

修改数据库结构前，需要先在小组群中说明，避免前后端接口和数据库字段不一致。
