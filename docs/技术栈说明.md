# FlavorLog 技术栈说明

## 一、总体技术路线

FlavorLog 采用前后端分离架构。

```text
前端：Flutter 移动端 App
后端：FastAPI RESTful API
数据库：PostgreSQL
部署：Docker Compose 本地启动数据库
联调方式：局域网 C/S 模拟
```

阶段一主要目标是完成 MVP 核心业务功能，并在局域网环境下完成手机端与后端服务联调。

---

## 二、前端技术栈

## 1. Flutter

### 用途

用于开发移动端 App。

### 主要负责内容

- 页面 UI。
- 页面跳转。
- 移动端状态管理。
- 摄像头调用。
- 麦克风调用。
- 本地数据存储。
- 调用后端接口。

### 建议版本

```text
Flutter stable
Dart stable
```

---

## 2. GetX

### 用途

用于 Flutter 状态管理和路由管理。

### 主要负责内容

- 全局状态管理。
- 页面局部状态管理。
- 登录状态保存。
- 多账号切换时状态清理。
- 页面路由跳转。

---

## 3. Dio

### 用途

用于前端发送 HTTP 请求。

### 主要负责内容

- 封装 GET / POST / PUT / DELETE 请求。
- 统一配置后端 API 地址。
- 统一处理请求头。
- 统一处理错误响应。
- 上传图片、视频、音频文件。

---

## 4. SQLite

### 用途

用于前端本地数据存储。

### 主要负责内容

- 本地缓存用户数据。
- 本地缓存饮食记录。
- 断网情况下临时保存数据。
- 按 user_id 隔离不同账号数据。

---

## 5. Camera / Microphone API

### 用途

用于采集图片、视频和音频数据。

### 主要负责内容

- 拍摄食物图片。
- 录制饮食视频。
- 采集语音描述。
- 将采集的数据上传到后端。

## 三、后端技术栈

## 1. Python 3.11

### 用途

后端主要开发语言。

### 说明

所有后端成员建议使用 Python 3.11，避免由于版本差异导致依赖安装失败或运行结果不一致。

---

## 2. FastAPI

### 用途

用于构建后端 RESTful API。

### 主要负责内容

- 用户注册登录接口。
- 饮食记录接口。
- 赛博冰箱接口。
- 健康反馈接口。
- 社区动态接口。
- 推荐算法接口。
- 文件上传接口。

- 管理数据库会话。
- 执行数据库增删改查。

## 5. Pydantic

### 用途

用于接口请求和响应数据校验。

### 主要负责内容

## 四、数据库技术栈

## 1. PostgreSQL 16

### 用途

项目主数据库。

### 主要存储内容

- 用户信息。
- 饮食记录。
- 冰箱食材。
- 健康反馈。
- 社区动态。
- 评论信息。
- 点赞信息。
- 推荐结果。
- 用户口味向量。

---

## 2. Docker Compose

### 用途

统一启动 PostgreSQL 数据库环境。

### 启动命令

```bash
docker compose up -d
```

### 停止命令

```bash
docker compose down
```

### 重置数据库

```bash
docker compose down -v
docker compose up -d
```

## 五、AI 与算法技术栈

## 1. OpenCV

### 用途

用于图像和视频处理。

### 主要负责内容

- 视频分镜切分。
- 关键帧抽取。
- 图像清晰度判断。
- 食物图像预处理。

---

## 2. YOLOv8

### 用途

用于食物或食材目标检测。

### 主要负责内容

- 识别图片或视频帧中的食物区域。
- 返回食物目标框。
- 为后续热量估算或食材识别提供基础结果。

---

## 3. OCR

### 用途

用于识别菜单或图片中的文字。

### 主要负责内容

- 菜单文字提取。
- 外食场景下的菜品识别。
- 为 LLM 健康分析提供文本输入。

---

## 4. LLM API

### 用途

用于生成饮食建议和食谱推荐。

### 主要负责内容

- 根据冰箱库存生成食谱。
- 根据用户健康目标给出饮食建议。
- 对菜单进行健康红绿灯标注。
- 对社区内容进行 AI 点评。

---

## 5. Pandas

### 用途

用于数据处理和 Mock 数据生成。

### 主要负责内容

- 处理用户历史饮食数据。
- 生成演示数据。
- 统计健康反馈。
- 辅助算法分析。

---

## 6. Scikit-learn

### 用途

用于基础机器学习和相似度计算。

### 主要负责内容

- 用户口味向量计算。
- 余弦相似度匹配。
- 简单聚类或推荐逻辑。
- 阶段一小样本匹配算法。

## 八、依赖管理规范

### 后端依赖

后端依赖统一写入：

```text
backend/requirements.txt
```

安装命令：

```bash
pip install -r requirements.txt
```

### 前端依赖

前端依赖统一写入：

```text
frontend/pubspec.yaml
```

安装命令：

```bash
flutter pub get
```

## 九、环境变量管理规范

真实环境变量文件不提交到 GitHub。

需要提交的是模板文件：

```text
.env.example
backend/.env.example
```

本地开发时，每个人自行复制：

```powershell
copy .env.example .env
copy backend\.env.example backend\.env
```
