# 知味志 FlavorLog 项目规划

## 1. 阶段一目标

阶段一目标是完成局域网 MVP 演示，重点跑通核心业务闭环，不追求完整高并发架构。

阶段一必须完成：

1. 用户注册与登录。
2. 饮食记录新增与查询。
3. 图片、视频、音频上传的基础流程。
4. 赛博冰箱食材新增与查询。
5. Mock 食谱推荐。
6. 健康反馈与 Mock 红黑榜。
7. 社区动态发布与查询。
8. 局域网环境下前后端联调。

## 2. 阶段二目标

阶段二在阶段一基础上进行架构演进，包括：

1. Redis 缓存。
2. Celery 异步任务。
3. RabbitMQ 或 Redis List 消息队列。
4. PgBouncer 数据库连接池。
5. WebSocket 实时推送。
6. 高并发压测。

## 3. MVP 功能边界

阶段一允许以下模块先使用 Mock：

1. 图片识别。
2. 视频极速录入。
3. OCR 菜单扫描。
4. LLM 食谱生成。
5. 食物红黑榜。
6. 口味搭子匹配。

但 Mock 返回格式必须严格遵守 `docs/接口文档.md`。

## 4. 第一周任务

1. 完成环境配置。
2. 完成前端基础骨架。
3. 完成后端基础启动。
4. 完成数据库初始化。
5. 完成注册登录接口。
6. 完成前端登录页面和首页。

## 5. 第二周任务

1. 完成饮食记录接口。
2. 完成饮食记录页面。
3. 完成文件上传接口。
4. 完成赛博冰箱食材 CRUD。
5. 完成赛博冰箱页面。

## 6. 第三周任务

1. 完成健康反馈接口。
2. 完成健康报告页面。
3. 完成社区动态接口。
4. 完成社区页面。
5. 完成 Mock 数据脚本。

## 7. 第四周任务

1. 完成全链路联调。
2. 完成局域网真机演示。
3. 修复主要 Bug。
4. 准备演示数据。
5. 准备课堂汇报材料。

# 知味志 FlavorLog 演示说明

本文档用于课堂演示前准备和现场演示流程说明。

## 1. 演示环境

| 项目     | 说明                        |
| -------- | --------------------------- |
| 前端     | Flutter App                 |
| 后端     | FastAPI                     |
| 数据库   | PostgreSQL 16               |
| 部署方式 | 局域网 C/S                  |
| 演示设备 | 一台电脑 + 一台手机或模拟器 |

## 2. 演示前准备

### 2.1 启动数据库

在项目根目录执行：

```bash
docker compose up -d
```
