# FlavorLog 知味志项目分工安排

## 一、团队分工原则

本项目采用前后端分离开发模式，阶段一重点完成 MVP 核心业务功能，并在局域网环境下实现前后端联调演示。

团队整体分为：

```text
前端开发组：2 人
后端开发组：4 人
```

其中，后端开发组包含：

```text
FastAPI 后端接口
PostgreSQL 数据库
AI 多模态算法
LLM 推荐算法
数据挖掘与社交算法
Mock 演示数据
```

## 四、公共协作要求

### 1. 开发前要求

每次开发前先拉取最新代码：

```bash
git pull
```

### 2. 提交前要求

提交前先查看修改：

```bash
git status
```

确认无误后提交：

```bash
git add .
git commit -m "类型: 修改说明"
git push
```

### 3. 不允许提交的内容

以下内容不得提交到 GitHub：

```text
.env
backend/.env
.venv/
真实 API Key
真实数据库密码
真实上传的图片、视频、音频文件
大型模型文件
临时缓存文件
```

### 4. 修改公共配置文件要求

修改以下文件前，需要先在小组群中说明：

```text
README.md
docker-compose.yml
.env.example
backend/.env.example
backend/requirements.txt
docs/接口文档.md
docs/数据库设计.md
```
